## 📝 部署步骤（下次部署参考）

### 1. 从 GitHub 下载代码

```bash
cd /var/www/gdxcsh-admin
curl -sL https://github.com/xiaoyaoluozi/gdxcsh-admin/raw/main/app.py -o app.py
curl -sL https://github.com/xiaoyaoluozi/gdxcsh-admin/raw/main/index.html -o index.html
```

### 2. 安装依赖

```bash
pip3 install flask flask-cors -q
```

### 3. 初始化数据库（首次部署）

```bash
python3 init_db.py
```

### 4. 启动 Flask 服务

```bash
pkill -f "python3.*app.py"
sleep 1
nohup python3 app.py > /var/log/gdxcsh-admin.log 2>&1 &
```

### 5. 配置 Nginx（首次部署）

```bash
sudo tee /etc/nginx/conf.d/gdxcsh-admin.conf > /dev/null << 'EOF'
server {
    listen 8080;
    server_name _;
    root /var/www/gdxcsh-admin;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF
sudo nginx -t && sudo systemctl reload nginx
```

### 6. 开放安全组端口

在阿里云 ECS 控制台 → 安全组配置 → 添加入方向规则：
- 端口范围：`8080/8080`
- 授权对象：`0.0.0.0/0`

## ⚠️ 遇到的问题与解决方案

### 问题 1: 编辑功能无法加载数据
**现象**: 点击"编辑"按钮后模态框打开但表单为空

**原因**: 前端 `editEvent()` 和 `editCompany()` 函数只打开了模态框，没有调用 API 加载数据

**解决**: 
1. 前端添加异步加载逻辑：
```javascript
async function editEvent(id) {
    const res = await fetch(`/api/events/${id}`);
    const data = await res.json();
    // 填充表单字段
}
```

2. 后端添加详情 API：
```python
@app.route('/api/events/<int:id>', methods=['GET'])
def get_event(id):
    # 返回单个活动详情
```

### 问题 2: GitHub 下载文件为空
**现象**: `curl` 下载的文件大小为 0 字节

**原因**: GitHub raw 链接有时不稳定

**解决**: 使用 GitHub 永久链接（带 commit hash）：
```bash
curl -sL https://github.com/xiaoyaoluozi/gdxcsh-admin/raw/c330cd0/app.py -o app.py
```

### 问题 3: 数据库表结构不匹配
**现象**: `sqlite3.OperationalError: table news has no column named summary`

**原因**: 旧数据库表结构缺少新字段

**解决**: 删除旧数据库重新初始化：
```bash
rm /var/www/gdxcsh-admin/data/admin.db
python3 init_db.py
```
